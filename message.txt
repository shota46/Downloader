#### 4.3 Routes -- YouTube

**GET `/api/youtube/info`** -- Fetch video metadata without downloading.

Query params: `url` (the YouTube URL).

This calls `yt_dlp.YoutubeDL` with `extract_info(url, download=False)` to retrieve title, thumbnail, duration, and available formats. The response is filtered to return only useful format info (resolution, file size, format_id, ext, whether it has video+audio or just one).

```python
@app.route('/api/youtube/info')
def youtube_info():
    url = request.args.get('url')
    if not url:
        return jsonify({'error': 'URL required'}), 400

    ydl_opts = {'quiet': True, 'no_warnings': True}
    try:
        with yt_dlp.YoutubeDL(ydl_opts) as ydl:
            info = ydl.extract_info(url, download=False)

        # Filter and simplify format list
        formats = []
        for f in info.get('formats', []):
            formats.append({
                'format_id': f.get('format_id'),
                'ext': f.get('ext'),
                'resolution': f.get('resolution', 'audio only'),
                'filesize': f.get('filesize') or f.get('filesize_approx'),
                'vcodec': f.get('vcodec'),
                'acodec': f.get('acodec'),
                'format_note': f.get('format_note', ''),
                'fps': f.get('fps'),
            })

        return jsonify({
            'title': info.get('title'),
            'thumbnail': info.get('thumbnail'),
            'duration': info.get('duration'),
            'uploader': info.get('uploader'),
            'formats': formats,
        })
    except Exception as e:
        return jsonify({'error': str(e)}), 500
```

**POST `/api/youtube/download`** -- Start a download in a background thread.

Request body (JSON): `url`, `quality` (one of: `"best"`, `"1080"`, `"720"`, `"480"`, `"360"`, `"audio"`).

This creates a task ID, launches a background thread that runs yt-dlp with a progress hook, and immediately returns the task ID so the frontend can poll for progress.

```python
@app.route('/api/youtube/download', methods=['POST'])
def youtube_download():
    data = request.json
    url = data.get('url')
    quality = data.get('quality', 'best')

    if not url:
        return jsonify({'error': 'URL required'}), 400

    task_id = str(uuid.uuid4())
    download_tasks[task_id] = {'status': 'starting', 'progress': 0, 'filename': None, 'error': None}

    thread = threading.Thread(target=_do_youtube_download, args=(task_id, url, quality))
    thread.daemon = True
    thread.start()

    return jsonify({'task_id': task_id})
```

The `_do_youtube_download` helper function:

```python
def _do_youtube_download(task_id, url, quality):
    format_map = {
        'best': 'bestvideo[ext=mp4]+bestaudio[ext=m4a]/best[ext=mp4]/best',
        '1080': 'bestvideo[height<=1080][ext=mp4]+bestaudio[ext=m4a]/best[height<=1080]',
        '720': 'bestvideo[height<=720][ext=mp4]+bestaudio[ext=m4a]/best[height<=720]',
        '480': 'bestvideo[height<=480][ext=mp4]+bestaudio[ext=m4a]/best[height<=480]',
        '360': 'bestvideo[height<=360][ext=mp4]+bestaudio[ext=m4a]/best[height<=360]',
        'audio': 'bestaudio[ext=m4a]/bestaudio',
    }

    def progress_hook(d):
        if d['status'] == 'downloading':
            total = d.get('total_bytes') or d.get('total_bytes_estimate') or 0
            downloaded = d.get('downloaded_bytes', 0)
            pct = (downloaded / total * 100) if total > 0 else 0
            download_tasks[task_id]['status'] = 'downloading'
            download_tasks[task_id]['progress'] = round(pct, 1)
        elif d['status'] == 'finished':
            download_tasks[task_id]['status'] = 'processing'
            download_tasks[task_id]['progress'] = 100

    ydl_opts = {
        'format': format_map.get(quality, format_map['best']),
        'outtmpl': os.path.join(YOUTUBE_DIR, '%(title)s.%(ext)s'),
        'progress_hooks': [progress_hook],
        'quiet': True,
        'merge_output_format': 'mp4' if quality != 'audio' else None,
    }
    if quality == 'audio':
        ydl_opts['postprocessors'] = [{'key': 'FFmpegExtractAudio', 'preferredcodec': 'mp3'}]

    try:
        with yt_dlp.YoutubeDL(ydl_opts) as ydl:
            info = ydl.extract_info(url)
            filename = ydl.prepare_filename(info)
            if quality == 'audio':
                filename = os.path.splitext(filename)[0] + '.mp3'
        download_tasks[task_id]['status'] = 'complete'
        download_tasks[task_id]['filename'] = os.path.basename(filename)
    except Exception as e:
        download_tasks[task_id]['status'] = 'error'
        download_tasks[task_id]['error'] = str(e)
```

**POST `/api/anime/download`** -- Download an HLS stream using ffmpeg.

This is the most involved anime route. It receives an HLS URL (the `.m3u8` link from the stream endpoint) and uses `ffmpeg` via subprocess to download and mux the stream into an MP4 file. Progress is tracked via a background thread similar to YouTube downloads.

```python
import subprocess
import re

@app.route('/api/anime/download', methods=['POST'])
def anime_download():
    data = request.json
    hls_url = data.get('url')
    filename = data.get('filename', 'anime_episode')
    referer = data.get('referer', 'https://megacloud.tv')

    if not hls_url:
        return jsonify({'error': 'HLS URL required'}), 400

    task_id = str(uuid.uuid4())
    download_tasks[task_id] = {'status': 'starting', 'progress': 0, 'filename': None, 'error': None}

    # Sanitize filename
    safe_name = re.sub(r'[^\w\s\-.]', '', filename).strip() + '.mp4'
    output_path = os.path.join(ANIME_DIR, safe_name)

    thread = threading.Thread(target=_do_anime_download, args=(task_id, hls_url, output_path, safe_name, referer))
    thread.daemon = True
    thread.start()

    return jsonify({'task_id': task_id})
```

The `_do_anime_download` helper:

```python
def _do_anime_download(task_id, hls_url, output_path, filename, referer):
    try:
        download_tasks[task_id]['status'] = 'downloading'
        cmd = [
            'ffmpeg', '-y',
            '-headers', f'Referer: {referer}\r\n',
            '-i', hls_url,
            '-c', 'copy',
            '-bsf:a', 'aac_adtstoasc',
            output_path
        ]
        process = subprocess.Popen(cmd, stderr=subprocess.PIPE, universal_newlines=True)

        # ffmpeg outputs progress to stderr; parse duration and time for progress
        duration = None
        for line in process.stderr:
            # Parse duration from stream info
            dur_match = re.search(r'Duration:\s*(\d+):(\d+):(\d+)', line)
            if dur_match:
                h, m, s = int(dur_match.group(1)), int(dur_match.group(2)), int(dur_match.group(3))
                duration = h * 3600 + m * 60 + s

            # Parse current time
            time_match = re.search(r'time=(\d+):(\d+):(\d+)', line)
            if time_match and duration and duration > 0:
                h, m, s = int(time_match.group(1)), int(time_match.group(2)), int(time_match.group(3))
                current = h * 3600 + m * 60 + s
                pct = min(100, round(current / duration * 100, 1))
                download_tasks[task_id]['progress'] = pct

        process.wait()
        if process.returncode == 0:
            download_tasks[task_id]['status'] = 'complete'
            download_tasks[task_id]['filename'] = filename
            download_tasks[task_id]['progress'] = 100
        else:
            download_tasks[task_id]['status'] = 'error'
            download_tasks[task_id]['error'] = 'ffmpeg exited with non-zero status'
    except Exception as e:
        download_tasks[task_id]['status'] = 'error'
        download_tasks[task_id]['error'] = str(e)
```

   - **Download**: Calls `POST /api/anime/download` with the HLS URL and a generated filename. Shows progress bar, polls for completion.

Key code flow:

```javascript

    const resp = await fetch(`/api/anime/search?keyword=${encodeURIComponent(keyword)}&page=${page}`);
    const data = await resp.json();
    renderSearchResults(data.data.response, data.data.pageInfo);
}

async function showAnimeDetail(animeId) {
    const [infoResp, epsResp] = await Promise.all([
        fetch(`/api/anime/info/${animeId}`),
        fetch(`/api/anime/episodes/${animeId}`)
    ]);
    const info = await infoResp.json();
    const eps = await epsResp.json();

async function getStream(episodeId, server, type) {
    const resp = await fetch(`/api/anime/stream?id=${encodeURIComponent(episodeId)}&server=${server}&type=${type}`);
    const data = await resp.json();
    // data.data[0].link.file => HLS URL

    renderStreamOptions(data.data[0]);
```

### 7. Key Implementation Details

**Episode ID Encoding**: The hianime-API uses episode IDs like `steinsgate-3::ep=213`. The `::` contains a colon which can cause routing issues. Using Flask's `<path:episode_id>` converter handles this correctly. On the frontend, the episode ID must be URL-encoded when passed as a query parameter but can be passed raw in path segments.

**HLS Download via ffmpeg**: The stream endpoint returns M3U8 (HLS) links, not direct MP4 files. To download these as video files, we use `ffmpeg -i <m3u8_url> -c copy output.mp4`. The `-c copy` flag avoids re-encoding (fast). The `Referer` header must be set to `https://megacloud.tv` (as seen in the stream extract code at `/Users/hiro/Documents/Downloader/hianime-API/src/modules/stream/stream.extract.js` line 10 and 27).

**Progress Tracking**: Both YouTube and anime downloads use the same `download_tasks` dictionary and the same progress polling endpoint. The task_id is a UUID. Tasks are kept in memory (no database needed for a local tool). A cleanup could be added to remove old tasks, but for a local tool this is not necessary.

**CORS is Not an Issue**: Since both the frontend and the API are served from the same Flask origin (`localhost:5000`), there are no CORS concerns. The hianime-API at port 3030 is only accessed server-side by Flask, never directly from the browser.

**ffmpeg Requirement**: The anime download feature and yt-dlp's format merging both require `ffmpeg` to be installed on the system. This should be documented. On macOS: `brew install ffmpeg`.

### 10. Potential Challenges

- **yt-dlp format selection**: The format strings can be finicky. The `bestvideo[height<=X]+bestaudio` pattern works well but falls back gracefully if a specific height is not available. Testing with various URLs (shorts, livestreams, age-restricted) is important.
- **HLS stream availability**: The hianime-API's stream decryption (megacloud parser) can break if the upstream site changes its encryption. This is inherent to the scraping approach and not something we can control.
- **ffmpeg not installed**: Must detect this at startup and show a clear error. Add a check in `app.py` startup: `subprocess.run(['ffmpeg', '-version'], capture_output=True)`.
- **Large files**: YouTube 1080p videos can be several GB. The download directory could grow. Adding a simple "delete" button in the Downloads tab would help.
- **Thread safety**: The `download_tasks` dictionary is accessed from multiple threads. For a local single-user tool, Python's GIL makes simple dict operations safe enough. No mutex needed for this use case.
